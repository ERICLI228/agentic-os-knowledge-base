## 4. AutoResearch 分析

**项目**: [karpathy/autoresearch](https://github.com/karpathy/autoresearch)
- **Stars**: 69,701 ⭐
- **语言**: Python (83.4%), Jupyter Notebook (16.6%)
- **创建时间**: 2026-03-06

### 核心理念
让 AI 代理在单个 GPU 上自主运行 LLM 训练实验。AI 修改代码、训练5分钟、检查结果、保留或丢弃，然后重复。

### 核心文件
- `prepare.py` — 固定常量，数据准备（不修改）
- `train.py` — 模型、优化器、训练循环（AI 修改此文件）
- `program.md` — AI 代理的指令（人类修改）

### 关键设计
- **固定时间预算**: 训练总是恰好运行5分钟
- **评估指标**: val_bpb (validation bits per byte) — 越低越好
- **实验速度**: 约 12 次实验/小时，睡眠中约 100 次

### 可借鉴点
1. **program.md 技能系统** — 类似 OpenClaw 的 skills
2. **固定时间预算** — 可用于我们的任务执行
3. **自进化训练** — AI 自主优化模型

### 对我们的价值
⭐⭐⭐⭐☆ (4/5)
- 与我们的目标（AI 自动科研）高度相关
- program.md 机制可借鉴到 skill 系统

## 5. Hermes Agent vs OpenClaw 对比

**项目**: [NousResearch/hermes-agent](https://github.com/NousResearch/hermes-agent)
- **Stars**: 47,182 ⭐
- **创建时间**: 2025-07-22
- **最新版本**: v2026.4.8 (2026-04-08)

### 核心特性对比

| 特性 | Hermes Agent | OpenClaw |
|------|-------------|----------|
| 内置学习循环 | ✅ 有 | ❌ 无 |
| 技能自进化 | ✅ 使用时自动改进 | ❌ 静态 |
| 记忆系统 | ✅ FTS5 会话搜索 + LLM 摘要 | ✅ L1-L4 分层 |
| 平台支持 | Telegram/Discord/Slack/WhatsApp/Signal/CLI | 多平台 |
| 定时任务 | ✅ 内置 cron + 自然语言 | ✅ Cron |
| MCP 集成 | ✅ 支持 | ✅ 支持 |
| OpenClaw 迁移 | ✅ `hermes claw migrate` | — |
| 用户建模 | ✅ Honcho dialectic | ❌ 无 |

### 差异化优势

**Hermes Agent 独有:**
1. **自进化能力** — 技能在使用中自动改进
2. **封闭学习循环** — Agent 自主记忆和知识持久化
3. **多平台消息网关** — 单一进程支持 Telegram/Discord/Slack/WhatsApp/Signal
4. **语音备忘录转录** — 跨平台对话连续性
5. **Atropos RL** — 研究级轨迹生成
6. **服务器less** — Daytona/Modal 支持，闲置时休眠

**OpenClaw 独有:**
1. **飞书集成** ✅ — 国内生态优势
2. **8 群日报系统** ✅ — 自动化运营
3. **TK 运营自动化** ✅ — 东南亚电商
4. **三备份系统** ✅ — 数据安全

### 迁移支持
Hermes 提供 `hermes claw migrate` 命令，可自动导入：
- SOUL.md
- 记忆 (MEMORY.md, USER.md)
- Skills → `~/.hermes/skills/openclaw-imports/`
- 命令 allowlist
- 消息设置
- API keys
- TTS 资源
- AGENTS.md

### 评估
- 如果需要更强自进化能力 → 考虑迁移 Hermes
- 如果需要飞书/TK运营 → 继续使用 OpenClaw
- **可考虑双跑**: OpenClaw 做 TK 运营，Hermes 做开发
